# 🔹 1. Using **kubectl** (Cluster exploration from CLI)

First, connect to the AKS cluster:

```bash
az aks get-credentials --resource-group myResourceGroup --name myAKSCluster
```

Now explore:

### 🖥️ Cluster & Nodes

* Show cluster info:

  ```bash
  kubectl cluster-info
  ```
* List worker nodes:

  ```bash
  kubectl get nodes -o wide
  ```

### 📦 Namespaces & Workloads

* Show namespaces:

  ```bash
  kubectl get ns
  ```
* Check pods (default namespace):

  ```bash
  kubectl get pods
  ```
* Check system pods (Kube + Azure addons):

  ```bash
  kubectl get pods -n kube-system
  ```
* See deployments:

  ```bash
  kubectl get deployments
  ```

### 🔎 Deep Dive

* Pod details:

  ```bash
  kubectl describe pod <pod-name>
  ```
* Logs of a pod:

  ```bash
  kubectl logs <pod-name>
  ```
* Events for troubleshooting:

  ```bash
  kubectl get events --sort-by=.metadata.creationTimestamp
  ```

---

# 🔹 2. Using **Azure Management Console (Portal)**

1. Go to [Azure Portal](https://portal.azure.com).
2. Navigate to **Kubernetes services** → select your AKS cluster.

### Key Areas to Explore:

* **Overview** → Cluster name, region, Kubernetes version, API endpoint.
* **Node pools** → VM size, node count, auto-scaling.
* **Workloads** → Deployments, Pods, ReplicaSets (if monitoring is enabled).
* **Services and Ingress** → Load Balancers, DNS, Ingress Controllers.
* **Insights (Monitoring)** → CPU, memory, pod usage, container logs.
* **Access control (IAM)** → RBAC + Azure AD integration.
* **Networking** → VNet, subnets, DNS, Load Balancers.
* **Upgrades** → Update Kubernetes version or scale node pools.

---

# 🔹 3. How They Complement Each Other

* **kubectl** → Best for **real-time debugging, deployment, and pod-level details**.
* **Azure Portal** → Best for **infrastructure, scaling, monitoring dashboards, and identity/security settings**.
* Together, they give you **full control and visibility** into your AKS cluster.
